# MS-Rewards-Automator 用户指南

## 快速开始

### 环境准备

```bash
git clone https://github.com/Disaster-Terminator/ms-rewards-automator.git
cd ms-rewards-automator
conda env create -f environment.yml
conda activate ms-rewards-bot
playwright install chromium
```

### 首次运行

```bash
# 复制配置文件
cp config.example.yaml config.yaml
# 编辑 config.yaml，填写账号信息

# 运行
python main.py
```

**登录方式**：

- **手动登录**（推荐）：首次运行时浏览器打开，手动登录后保存会话
- **自动登录**：在 `config.yaml` 的 `auto_login` 部分填写账号密码和 2FA 密钥

## Tauri 桌面应用

### 安装

下载发布页面的安装包（MSI 或 NSIS），双击安装即可。

### 启动

双击桌面图标或开始菜单中的 "MS Rewards Automator" 启动应用。

应用会自动：
1. 启动 Python 后端服务（动态端口）
2. 加载前端界面
3. 建立 WebSocket 实时连接

### 界面功能

#### 仪表盘

- **当前积分**：显示账户当前积分和今日获得积分
- **任务进度**：显示桌面搜索（默认30次）和移动搜索（默认20次）完成进度
- **运行时间**：显示脚本运行时长
- **任务提示**：显示未完成任务、错误、积分获取失败等提示信息
- **后端状态**：显示后端连接状态和心跳检测

#### 任务控制

- **执行模式**：
  - 正常模式：30+20 搜索，完整功能
  - 用户模式：3+3 搜索，验证稳定性
  - 开发模式：2+2 搜索，快速调试
- **运行选项**：
  - 无头模式：不显示浏览器窗口
  - 仅桌面搜索：只执行桌面搜索
  - 仅移动搜索：只执行移动搜索
  - 跳过每日任务：跳过每日任务系统
- **启动/停止**：点击按钮控制任务执行

#### 配置管理

- 搜索设置（搜索次数、间隔时间）
- 浏览器设置（类型、无头模式）
- 登录设置（自动登录、2FA）
- 通知设置（Telegram）
- 调度器设置
- 监控设置

#### 日志查看

- 实时日志流
- 按级别过滤（INFO/WARNING/ERROR/DEBUG）
- 关键词搜索
- 导出日志

#### 历史记录

- 执行历史列表
- 积分趋势图表
- 统计数据（总积分、日均、搜索次数）

### 主题切换

点击右上角的主题切换按钮可在暗色模式和亮色模式之间切换。

### 视觉效果

Windows 11 用户可享受 Mica 毛玻璃效果，窗口具有透明背景和现代视觉体验。

## 开发模式

### 启动开发环境

```bash
# 1. 启动 Python 后端
conda activate ms-rewards-bot
python web_server.py --host 127.0.0.1 --port 8000

# 2. 启动 Tauri 开发模式
cd frontend
npm install
npm run tauri:dev
```

开发模式下：
- 后端使用固定端口 8000
- 前端通过 Vite 热更新
- 支持 React DevTools 调试

### 构建生产版本

```bash
cd frontend
npm run tauri:build
```

构建产物位于 `src-tauri/target/release/bundle/` 目录。

## 核心配置

编辑 `config.yaml`：

```yaml
search:
  desktop_count: 30
  mobile_count: 20
  wait_interval: { min: 5, max: 15 }

browser:
  headless: false
  slow_mo: 100

login:
  state_machine_enabled: true
  timeout_seconds: 300

task_system:
  enabled: true

notification:
  enabled: false
```

## 登录配置

### 自动登录

```yaml
login:
  auto_login:
    enabled: true
    email: "your-email@example.com"
    password: "your-password"
    totp_secret: "JBSWY3DPEHPK3PXP"  # 2FA 密钥
```

获取TOTP密钥：在Microsoft账户安全设置中启用2FA → 选择"使用身份验证器应用" → 点击"无法扫描？" → 复制显示的密钥。

---

## 任务系统

```yaml
task_system:
  enabled: true
  min_delay: 2
  max_delay: 5
  skip_completed: true
```

支持的任务类型：

- **URL Reward**: 访问链接获取积分
- **Quiz**: 问答任务（随机选择答案）
- **Poll**: 投票任务

## 通知配置

### Telegram

```yaml
notification:
  enabled: true
  telegram:
    enabled: true
    bot_token: "你的Bot Token"
    chat_id: "你的Chat ID"
```

获取方式：@BotFather 创建机器人获取token，@userinfobot 获取Chat ID。

### Server酱（微信推送）

```yaml
notification:
  serverchan:
    enabled: true
    key: "你的SendKey"
```

获取方式：访问 <https://sct.ftqq.com/> 微信扫码登录。

## 故障排除

### 常见问题

| 问题 | 解决方案 |
|------|----------|
| 无法连接Microsoft服务 | 检查网络，WSL2用户建议在Windows运行 |
| 登录超时 | 增加 `timeout_seconds` 到 600 |
| 积分未增加 | 检查是否已达每日上限，查看日志 |
| TOTP验证失败 | 检查密钥正确性，同步系统时间 |
| 浏览器启动失败 | 运行 `playwright install chromium` |
| 后端连接失败 | 检查后端服务是否启动，查看控制台日志 |

### 日志位置

- 日志文件：`logs/automator.log`
- 调试模式：`python main.py --dev`

## 安全建议

1. **保护配置文件**：`config.yaml` 包含敏感信息，已在 `.gitignore` 中
2. **不要提交敏感信息**：确保不要将 `config.yaml` 提交到版本控制
3. **定期更换密码**

---

## 最佳实践

- 使用慢速模式：`python main.py --mode slow`
- 启用调度器随机时间运行
- 不要短时间内多次运行
- 定期检查账号状态
